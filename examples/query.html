Query — a PodOS page, running real SPARQL through Comunica

A use case: PodOS has no SPARQL or triple-pattern abilities. Sol Components supports both. A PodOS app wants to gain those abilities but not use Sol Component components. Component Interop to the reqcue: if a manifest makes an ability available (which Sol's does), the ability is available to the app just by refering to its manifest and using it in a page. OTOH, importing the component requires a specific data-components listing. So we'll import just the SPARQL capability in the call to component-intero like this:

A <ul> inside <pos-app> querying DBpedia for Turing Award recipients

     
        // we load the PodOS and Sol manifests
    data-manifest="sol-components.manifest.json pod-os.manifest.json"

    // we load the PodOS components BUT NOT the Sol components
    data-components="@pod-os/elements"                       

    <script src="component-interop.js"
        data-stage="auto" data-manifest-default="off"
        data-manifest="sol-components.manifest.json pod-os.manifest.json"
        data-components="@pod-os/elements"
></script>

<!-- A plain <ul> inside the PodOS app gains SPARQL. data-from-query runs the
     query through Comunica against DBpedia and renders one <li> per row straight
     into this <ul> (no view — the container's tag decides the shape); {{award}} is
     filled from data-var-award. -->
<pos-app>
  <ul data-from-query
      data-endpoint="https://dbpedia.org/sparql"
      data-sparql="SELECT DISTINCT ?name WHERE {
                ?p <http://dbpedia.org/ontology/award> <http://dbpedia.org/resource/{{award}}> ;
                   <http://www.w3.org/2000/01/rdf-schema#label> ?name .
                FILTER(lang(?name) = 'en')
              } ORDER BY ?name LIMIT 5"
      data-var-award="Turing_Award"
  ></ul>
</pos-app>
