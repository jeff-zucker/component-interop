Shared Navigation — open a resource with one library, both libraries follow

`component-interop` loads Solid Web Components and PodOS and wires their resource channels together — no glue code. Browse on the left; PodOS on the right follows. Open a folder in the Solid Web Components browser and PodOS shows that same folder.

Browse here — Solid Web Components

<sol-pod> file browser. Click a folder to open it.

Follows along — PodOS

<pos-*> shows whatever you open on the left.

<!-- USE COMPONENT-INTEROP TO LOAD TWO SETS OF COMPONENTS
-->
<script src="component-interop.js"
        data-stage="auto" data-manifest-default="off"
        data-manifest="sol-components.manifest.json pod-os.manifest.json"
        data-components="sol-pod @pod-os/elements"
></script>

<!-- PUT THE GUEST COMPONENT'S HTML IN THE PAGE
-->
<sol-pod source="https://timbl.solidcommunity.net/public/"></sol-pod>

<!-- PodOS follows the same current resource via the broker's resource channel -->
<pos-app>
  <pos-resource>
    <pos-container-contents></pos-container-contents>
  </pos-resource>
</pos-app>

# THE GUEST LIBRARY DECLARES IN ITS MANIFEST THAT IT PROVIDES SOMETHING
# -- swc says it sends the url from a sol-navigate event under the key `navigation`
#
{
  "objects": {
    "provides": { "navigation": { "respondTo": "sol-navigate", "sendValue": "detail.url" } }
  }
}

# THE HOST LIBRARY DECLARES IN ITS MANIFEST THAT IT ACCEPTS THAT THING
# -- PodOS accepts `navigation` and slots the url it gets into pos-resource's uri attribute
#
{
  "objects": {
    "accepts": { "navigation": { "onElement": "pos-resource", "applyValueTo": "uri" } }
  }
}
