Forms — a PodOS page, gaining Solid Web Components editing

PodOS displays a resource. Because the data-edit-* attributes below auto-load Solid Web Components' editing modules, a plain <div> next to it becomes an editor generated from a SHACL shape — data-edit-shape + data-subject + data-edit-mode, no component code. swc and PodOS read the same RDF; the form edits the very data PodOS shows.

PodOS shows the resource

Solid Web Components edits it — a form from a shape

The plain <div> never had to be a component — the capability supplied the behaviour, and <sol-settings> gathers the editor into a panel. (Saving needs write access; against this static sample it renders and edits the form — the PUT is what a real pod would accept. The Auth tab supplies the session that makes a save land.) 

# sol-components manifest — the data-edit-* attributes auto-load a bundle
# (no object shared here — using the attributes just pulls in code on demand)
{
  "bundles": {
    "rdf-bundle": ["solid-logic", "solid-ui", "sol-form", "sol-settings", "…"]
  },
  "attributes": {
    "data-edit-shape data-subject data-edit-mode": "rdf-bundle"
  }
}
