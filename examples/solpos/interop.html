What Component Interop provides

While component-interop's main function is brokering between components as shown in the other tabs, it can also be used with a single component library. By invoking the component-interop script you get these things regardless of whether you load other component libraries or not.

  1. a manifest driven loader (user knows the URL of the manifest, not of individual components or prereqs)
  2. a single attribute that can be used on any element on any page: data-handler
  3. a single named emited signal 'interop:activate' which can be listened for when using data-handler

The picker at right uses the data-handler attribute on ordinary buttons to create a tabset that display the PodOS component pos-image.

data-handler supports directly displayhing components based on clicks in menus, tabs, selectors, and buttons.
