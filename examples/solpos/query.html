Query — a PodOS page, gaining a Solid Web Components capability

PodOS has no SPARQL view. But because component-interop also loaded Solid Web Components' sparql capability, any element on this page can run a query. Below, PodOS shows a folder its own way; the plain <ul data-from-query> next to it lists the same folder with SPARQL — one attribute, no extra component.

PodOS

A plain element with the swc sparql capability

    <script src="component-interop.js"
        data-stage="local"
        data-manifest="sol-loader.manifest.json pod-os.manifest.json"
        data-bundles="@pod-os/elements"
        data-extend-with="sparql"
></script>

<pos-app>
  <pos-resource uri="https://timbl.solidcommunity.net/public/">
    <pos-container-contents></pos-container-contents>
  </pos-resource>
</pos-app>

<!-- any element gains SPARQL via the data-from-query attribute -->
<ul data-from-query
    endpoint="https://timbl.solidcommunity.net/public/"
    pattern="?folder <http://www.w3.org/ns/ldp#contains> ?item"
    view="list"
></ul>

    # solid-web-components manifest — the sparql capability swc ships
# (no channel here — data-extend-with="sparql" just loads code)
{
  "capabilities": {
    "sparql": {
      "modules": ["solid-web-components/core/from-query.js"],
      "attributes": ["data-from-query"]
    }
  }
}
