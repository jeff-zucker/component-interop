Query — a PodOS page, running real SPARQL through Comunica

PodOS has no SPARQL view. But component-interop also loaded Solid Web Components' sparql capability, so any element inside this PodOS app can run a query — a real one, through Comunica (the engine the manifest pulls in on demand, so ORDER BY and LIMIT are honoured). The plain <ul> below sits inside <pos-app> and, via one data-from-query, asks the public DBpedia endpoint for programming languages. The class is a replaceable parameter — {{type}}, filled from var-type.

A <ul> inside <pos-app> querying DBpedia

     
    <script src="component-interop.js"
        data-stage="local"
        data-manifest="sol-loader.manifest.json pod-os.manifest.json"
        data-components="@pod-os/elements"
></script>

<!-- A plain <ul> inside the PodOS app gains SPARQL. data-from-query runs the
     query through Comunica against DBpedia; {{type}} is filled from var-type. -->
<pos-app>
  <ul data-from-query
      endpoint="https://dbpedia.org/sparql"
      sparql="SELECT DISTINCT ?name WHERE {
                ?thing a <http://dbpedia.org/ontology/{{type}}> ;
                       <http://www.w3.org/2000/01/rdf-schema#label> ?name .
                FILTER(lang(?name) = 'en')
              } ORDER BY ?name LIMIT 30"
      var-type="ProgrammingLanguage"
      view="list"
  ></ul>
</pos-app>
