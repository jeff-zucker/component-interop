Auth — one login, shared across two libraries

Sign in with Solid Web Components' <sol-login> (pick an issuer). On login this page reads your pod's /private/ two ways: swc's <sol-include> fetches it with the <sol-login> session, and PodOS's <pos-resource> is pointed at the same URL. The swc side proves the session is shared; whether the PodOS side picks it up is shown below.
Not signed in — pick an issuer and log in.

Solid Web Components

<sol-include raw> fetches /private/ with the sol-login session.

PodOS

<pos-resource> pointed at the same /private/.

Note: PodOS's <pos-app> manages its own session and exposes no hook to adopt an outside fetch, and swc's login patches rdflib's Fetcher (not global fetch). So the swc side is the reliable proof that the session is shared; the PodOS side may still prompt for its own login depending on how it fetches. component-interop wires what each library declares — it can't reach past an API a library doesn't offer. 

<script src="component-interop.js"
        data-stage="local"
        data-manifest="sol-loader.manifest.json pod-os.manifest.json"
        data-components="solid-web-components/core/inrupt-global.js sol-login sol-include @pod-os/elements"
></script>

<sol-login issuers="https://solidcommunity.net,https://solidweb.me,https://solidweb.org"></sol-login>

<sol-include id="swc-priv" raw></sol-include>
<pos-app><pos-resource id="pos-priv"></pos-resource></pos-app>

<!-- on login, point both at the pod's /private/ -->
<script type="module">
  document.addEventListener('sol-login', (e) => {
    const priv = new URL(e.detail.webId).origin + '/private/';
    document.getElementById('swc-priv').setAttribute('source', priv);
    document.getElementById('pos-priv').setAttribute('uri', priv);
  });
</script>

# solid-web-components manifest — sol-login is a plain component; it shares the session
{
  "components": { "sol-login": "../web/sol-login.js" },
  "objects": {
    "provides": { "auth": { "service": "auth", "sendValue": "fetch" } },
    "consumes": { "auth": { "call": "adoptFetch" } }
  }
}
