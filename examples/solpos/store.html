Shared Store — two libraries, one RDF graph

The broker pairs PodOS's provides: rdf with Solid Web Components' consumes: rdf.useStore: PodOS hands over its internalStore and swc adopts it, so both libraries read and write the same rdflib graph — no glue code. PodOS loads a resource below; the panel underneath reports the live graph as swc sees it.

<!-- Load swc's rdf core BEFORE PodOS so swc's rdf.useStore consumer is
     registered before pod-os:loaded fires -->
<script src="component-interop.js"
        data-stage="local"
        data-manifest="sol-loader.manifest.json pod-os.manifest.json"
        data-components="solid-web-components/core/rdf.js @pod-os/elements"
></script>

<!-- PodOS loads a resource into its rdflib store -->
<pos-app>
  <pos-resource uri="https://timbl.solidcommunity.net/profile/card"></pos-resource>
</pos-app>

<!-- swc has adopted that very graph (rdf.useStore): -->
<script type="module">
  const { store } = ComponentInterop.services.get('rdf');
  // `store` IS PodOS's internalStore — same object, same triples
</script>

# pod-os.manifest.json — PodOS hands over its store
{
  "objects": {
    "provides": { "store": { "respondTo": "pod-os:loaded", "sendValue": "detail.os.store.internalStore" } }
  }
}

# solid-web-components manifest — swc adopts it (calls its rdf.useStore handler)
{
  "objects": {
    "consumes": { "store": { "call": "rdf.useStore" } }
  }
}
