Shared Store — two libraries, one RDF graph

The broker pairs PodOS's provides: rdf with Solid Web Components' consumes: rdf.useStore: PodOS hands over its internalStore and swc adopts it, so both libraries read and write the same rdflib graph — no glue code. # pod-os.manifest.json — PodOS hands over its store
{
  "objects": {
    "provides": { "store": { "respondTo": "pod-os:loaded", "sendValue": "detail.os.store.internalStore" } }
  }
}

# sol-components manifest — swc adopts it (calls its rdf.useStore handler)
{
  "objects": {
    "consumes": { "store": { "call": "rdf.useStore" } }
  }
}
