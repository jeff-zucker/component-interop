component-interop — the broker, with two toy libraries

lib-a and lib-b are separate libraries that never import each other and share no glue script. lib-a's manifest says it provides a greeting capability (via its liba:hello event); lib-b's manifest says it consumes greeting (via a registered showGreeting handler). The broker pairs them.

Type in lib-a's input → lib-b's box updates. Zero dependencies, no bridge code.
